# Timeline

**Type:** Component
**Selector:** `<ui5-timeline>`
**Import:** `import { TimelineComponent } from '@ui5/webcomponents-ngx/fiori/timeline';`
**Export As:** `ui5Timeline`
**Package:** `@ui5/webcomponents-fiori` (fiori)

## Basic Usage
```html
<ui5-timeline [layout]="..." (ui5LoadMore)="onLoadMore($event)"></ui5-timeline>
```

## Description
The ui5-timeline component shows entries (such as objects, events, or posts) in chronological order. A common use case is to provide information about changes to an object, or events related to an object. These entries can be generated by the system (for example, value XY changed from A to B), or added manually. There are two distinct variants of the timeline: basic and social. The basic timeline is read-only, while the social timeline offers a high level of interaction and collaboration, and is integrated within SAP Jam.

| Name | Type | Default | Description |
|------|------|---------|-------------|
| `layout` | `TimelineLayout` | `"Vertical"` | Defines the items orientation. |
| `accessibleName` | `string | undefined` | `undefined` | Defines the accessible ARIA name of the component. |
| `loading` | `boolean` | `false` | Defines if the component should display a loading indicator over the Timeline. |
| `loadingDelay` | `number` | `1000` | Defines the delay in milliseconds, after which the loading indicator will show up for this component. |
| `growing` | `TimelineGrowingMode` | `"None"` | Defines whether the Timeline will have growing capability either by pressing a "More" button, or via user scroll. In bot |

> **IMPORTANT:** Use property bindings `[inputName]="value"`. Do NOT use `[attr.inputName]` — the ngx wrapper provides real Angular @Input()s.

## Outputs (Events)
| Angular Output (use this) | DOM Event (don't use) | Description |
|--------------------------|----------------------|-------------|
| `(ui5LoadMore)` | ~~`(load-more)`~~ | Fired when the user presses the More button or scrolls to the Timeline's end. **Note:** The event will be fired if growi |

> **IMPORTANT:** Always use the Angular output name (left column). Raw DOM event names will not trigger Angular change detection correctly with the ngx wrapper.

> **Event payload:** The ngx wrapper emits the `detail` object directly from the EventEmitter — access properties on the event itself (e.g. `event.selectedItems`), NOT via `event.detail.selectedItems`. The `.detail` wrapper is already unwrapped for you.
